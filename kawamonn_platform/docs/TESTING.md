# テスト手順書

## 1. 登録 & クォータテスト
1.  `https://register.kawamonn.com` にアクセスします。
2.  有効なメールアドレス（例: `test@example.com`）で登録します。
3.  `https://account.kawamonn.com` にログインします。
4.  クォータ（容量）が 20GB であることを確認します。
5.  ファイルマネージャ `https://web.kawamonn.com` に移動します。
6.  ファイルをアップロードし、成功することを確認します。
7.  残りのクォータを超えるサイズのファイルをアップロードしてみます（必要に応じて管理者側でクォータを小さくしてテスト）。
    *   *注*: クォータ制限を素早くテストするには、管理者に依頼してクォータを 10MB に下げてから、11MB のファイルをアップロードしてください。

## 2. 管理者 & セキュリティテスト
1.  `https://admin.kawamonn.com` にアクセスします。
2.  新しいユーザーが表示されていることを確認します。
3.  "Manage" をクリックします。
4.  **クォータ変更テスト**: クォータを 1GB に設定します。アカウントダッシュボードで反映されているか確認します。
5.  **パスワードリセットテスト**: "Generate Temp Password" を実行します。その一時パスワードでユーザーがログインできるか確認します。
6.  **削除テスト**: "Archive & Delete" を選択します。ユーザーディレクトリが `archives/` に移動され、DB ステータスが `deleted` になることを確認します。

## 3. 自動ライフサイクルテスト (シミュレーション)
"4年" は長いため、DB 操作で時間をシミュレートします:
1.  ユーザー `expiry_test@u-aizu.ac.jp` を作成します。
2.  手動で DB を更新します:
    ```sql
    UPDATE users SET expiry_at = datetime('now', '+2 days') WHERE username = 'expiry_test';
    ```
3.  Celery タスクを手動実行するか、スケジュールを待ちます:
    *   ログで "Reminder Sent" を確認します。
4.  DB を `expiry_at = datetime('now', '-1 day')` に更新します。
5.  クリーンアップタスクを待ちます:
    *   ユーザーが削除/アーカイブされたことを確認します。
    *   "Final Notice" がログに記録されていることを確認します。

## 4. メールテスト
1.  "Forgot Password" または "Reminder" をトリガーします（管理者画面の "Send Reminder" ボタンなど）。
2.  Mailgun のログまたは受信ボックスを確認します。
